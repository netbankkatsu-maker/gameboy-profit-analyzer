# Chrome Web Store 提出資料

## 基本情報

### 拡張機能名
**ゲーム買取利益判定ツール**

### 説明文（短）- 140字以内
Yahoo Fril のゲーム出品画像から自動でゲームを認識し、メルカリの売却相場を取得して利益を計算するツールです。

### 説明文（詳細）
```
ゲーム買取ビジネスをスムーズに！

このChrome拡張は以下の機能を提供します：

✨ 主な機能
• Yahoo Fril の出品画像から自動でゲームソフトを認識
• メルカリのリアルタイム相場データから単価を取得
• 購入金額との利益を自動計算
• リアルタイムダッシュボードで利益分析

🎮 対応機種
• ゲームボーイ
• ゲームボーイカラー
• ゲームボーイアドバンス
• その他レトログゲーム

📊 提供される情報
• 出品ごとの推定利益
• 利益率分析
• 過去30日の利益トレンド
• ゲーム別価格履歴

🔒 セキュリティ
• 最小限のパーミッション
• 個人情報の収集なし
• オフライン機能対応
• 定期自動更新

🚀 使い方
1. 拡張をインストール
2. Yahoo Fril で「ゲームボーイ」で検索
3. 拡張アイコンをクリック → 「今すぐ分析」
4. ダッシュボードで利益を確認
5. 定期自動分析（24時間ごと）

💡 このツールを使って、ゲーム買取ビジネスの効率化を実現しましょう！
```

## プライバシーポリシー

```
# ゲーム買取利益判定ツール プライバシーポリシー

最終更新: 2026年4月20日

## 1. 情報の収集

このChrome拡張は以下の情報を収集します：

### 収集する情報
- Yahoo Fril の出品ID、タイトル、価格
- メルカリの相場データ（公開情報）
- 分析結果（ローカルに保存）

### 収集しない情報
- 個人名、メールアドレス
- Yahoo Fril / Mercari のユーザーアカウント情報
- ブラウザ履歴、クッキー

## 2. 情報の使用

収集した情報は以下の目的でのみ使用されます：
- ゲーム認識精度の向上
- 相場データの更新
- ユーザー体験の改善

## 3. 情報の保存

- 分析結果はローカルストレージに保存
- クラウドへのアップロードなし
- ユーザーの明示的な同意がない限りデータ送信なし

## 4. セキュリティ

- すべての通信はHTTPS暗号化
- Firebase Firestore のセキュリティルール準拠
- 定期的なセキュリティ監査

## 5. 変更通知

ポリシー変更時は拡張の更新で通知されます。

## 6. 連絡先

質問やお問い合わせ：
support@gameboy-analyzer.example.com
```

## パーミッション説明

### storage
**理由**: 分析履歴と設定情報をローカルに保存するため
**使用方法**: chrome.storage.local で分析結果を永続化

### alarms
**理由**: 自動分析スケジュール（24時間ごと）を実行するため
**使用方法**: chrome.alarms.create() で定期実行を設定

### scripting
**理由**: Yahoo Fril ページでゲーム情報を抽出するため
**使用方法**: Content Script で出品情報を解析

### activeTab
**理由**: ユーザーがアクティブなページ（Yahoo Fril）にアクセスするため
**使用方法**: ポップアップから分析リクエストを送信

## 提出手順

1. **Chrome Web Store に開発者登録**
   - 5ドルの登録料を支払い
   - Google アカウントを確認

2. **拡張をアップロード**
   - manifest.json をzip形式で圧縮
   - Chrome Developer Dashboard にアップロード

3. **詳細情報を入力**
   - 上記の説明文、スクリーンショット、アイコン、ポリシーを入力

4. **レビュー待機**
   - 通常 2-5営業日
   - レビュー中に改善依頼が来る場合がある

5. **リリース**
   - 承認されたら自動で公開
   - ユーザーが即座にインストール可能

## トラブルシューティング

### レビュー却下時の対応

**よくある却下理由**:

1. **"Unnecessarily request broad permissions"**
   → 不要なパーミッションを削除
   → 必要最小限の host_permissions のみ設定

2. **"Injects ads or promotional content"**
   → 広告の埋め込みが疑われている
   → ポリシーを明確に説明

3. **"Deceptive functionality"**
   → 説明と実装が異なる
   → manifest.json と説明文を正確に一致させる

4. **"Security vulnerability"**
   → セキュリティ上の問題
   → CSP (Content Security Policy) を設定

**対応方法**:
- レビュー却下メッセージを詳読
- 問題を修正
- 再度アップロード（通常2-3回で承認）
